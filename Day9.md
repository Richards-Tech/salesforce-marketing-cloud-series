## 🚀 Journey Builder in SFMC

👉 Journey Builder is where data turns into communication  

---

### 🔹 What is Journey Builder?
Journey Builder is used to:  
- Automate customer communication  
- Trigger messages based on behavior/data  
- Design end-to-end customer journeys  

👉 It is the **automation + orchestration layer** of SFMC  

---

### 🔹 Why It Matters?
- Right message at the right time  
- Personalized experience  
- Fully automated campaigns  

👉 Without it:  
❌ Manual sends  
❌ No customer lifecycle automation  

---

### 🔹 Core Components

#### 🔹 Entry Source
👉 How contacts enter (Data Extension / API Event)  

#### 🔹 Decision Split
👉 Apply logic (Example: Active vs Inactive users)  

#### 🔹 Activities
👉 Actions:  
- Email  
- SMS  
- Update Data  

#### 🔹 Wait Activity
👉 Time control between steps  

📌 **Simple Flow:**  
Entry → Decision → Email → Wait → SMS → Exit  

---

### 🔹 Important Settings
✔ Re-entry mode (very important)  
✔ Entry filters  
✔ Exit criteria  

👉 Wrong setup = ❌ Journey not triggering  

---

### 🔹 How It Connects
👉 **Contact Builder** → Data source  
👉 **Journey Builder** → Logic  
👉 **Email/Mobile Studio** → Execution  

---

### 🔹 Use Cases
👉 Welcome journey  
👉 Abandoned cart  
👉 Reminder / follow-up  
👉 Re-engagement  

---

### 🔹 Best Practices
✅ Keep logic simple  
✅ Use proper entry conditions  
✅ Test with sample data  
✅ Monitor journey performance  

---

### 🔹 Common Issues
❌ Entry source mismatch  
❌ Duplicate contacts  
❌ Wrong filters  
❌ No re-entry allowed  

---

### 💡 Key Takeaway
**Journey Builder = Data + Logic + Timing**  
👉 If any one is wrong → Journey fails
