## 📱 Mobile Studio (Salesforce Marketing Cloud)

---

### 🔹 What is Mobile Studio?
Mobile Studio in Salesforce Marketing Cloud is used to send mobile-based communications such as:  
👉 SMS (Text Messages)  
👉 Push Notifications  
👉 In-App Messages  

💡 **Simple Understanding:**  
Mobile Studio = “Direct communication with customers on their mobile devices”  

---

### 🧠 Why Mobile Studio is Important?
Today, users check mobile faster than email.  
So Mobile Studio helps in:  
✔ Instant communication  
✔ Higher open rates  
✔ Real-time alerts & engagement  

---

### 🔧 Core Features of Mobile Studio

#### 🔹 1. SMS Messaging
Send text messages directly to customer mobile numbers.  
📌 Examples:  
- OTP / Verification codes  
- Appointment reminders  
- Order updates  

#### 🔹 2. Push Notifications
Send notifications to mobile apps.  
📌 Examples:  
- “Your flight is boarding now ✈️”  
- “Flash Sale starts now 🔥”  

#### 🔹 3. In-App Messaging
Messages shown inside mobile applications.  
📌 Examples:  
- Offers while using app  
- Feature announcements  

#### 🔹 4. Group Messaging (Bulk SMS)
Send messages to multiple users at once.  
📌 Example:  
- Promotional campaigns  
- Festival offers  

#### 🔹 5. Two-Way Messaging
Customers can reply to messages.  
📌 Example:  
- Reply “YES” to confirm appointment  
- Reply “STOP” to unsubscribe  

---

### 🔗 Integration with Journey Builder
Mobile Studio works with Journey Builder.  
👉 You can automate messages based on customer actions.  

📌 Example Journey:  
- User books flight → SMS confirmation  
- 24 hrs before → Reminder SMS  
- After travel → Feedback message  

---

### ⚙️ Key Components in Mobile Studio
| Component        | Purpose                          |
|------------------|----------------------------------|
| Contact Number   | Required for sending SMS         |
| Short Code/Long Code | Number used to send messages |
| Keyword          | Used for subscription (e.g., JOIN, STOP) |
| Message Template | Predefined SMS content           |
| Opt-in / Opt-out | Compliance & permission          |

---

### ⚠️ Important Rules
✔ Always take user consent (Opt-in) before sending SMS  
✔ Follow country regulations (like DND in India)  
✔ Keep messages short & clear  
✔ Include opt-out option (STOP)  

---

### 🎯 Real-Time Use Case
👉 Booking Done → SMS: “Your ticket is confirmed”  
👉 Before Flight → SMS Reminder  
👉 Delay → Instant Alert  
👉 After Flight → Feedback request  

---

### 💡 Key Takeaway
👉 Mobile Studio = Fastest way to reach customers  
👉 Works best for alerts, reminders, and urgent communication  
👉 Always combine with Journey Builder for automation
