## 🏗 SFMC Architecture (Studios vs Builders)

What I noticed while learning Salesforce Marketing Cloud:  
👉 Many people know the tools  
👉 But don’t understand how data, logic, and communication connect  

So let’s clarify this 👇

---

### 🔹 What is SFMC?
A platform to:  
- Store customer data  
- Automate communication  
- Send personalized messages  

---

### 🔹 Core Architecture
👉 **🔵 Studios = Execution (Communication tools)**  
👉 **🟢 Builders = Logic + Data (System brain)**  

---

### 🔵 STUDIOS (Execution Layer)
Used to send messages across channels:

- **Email Studio → Emails**  
  👉 Welcome, Promotions  

- **Mobile Studio → SMS / Push**  
  👉 OTP, Alerts  

- **Web Studio → CloudPages**  
  👉 Forms, Landing pages  

- **Advertising Studio → Ads**  
  👉 Retarget users  

- **Social Studio → Social engagement**  
  👉 Reply, Monitor  

---

### 🟢 BUILDERS (Configuration Layer)
Used to manage data & automation:

- **Contact Builder → Data model**  
  👉 Customer → Orders  

- **Journey Builder → Automation**  
  👉 Signup → Email → Reminder  

- **Automation Studio → Data processing**  
  👉 Import → Query → Update  

- **Content Builder → Templates**  
- **Audience Builder → Segmentation**  
- **Analytics Builder → Reports**  
- **Personalization Builder → AI recommendations**  

---

### 🔄 How It Works Together
👉 **Contact Builder** → Stores data  
👉 **Automation Studio** → Processes data  
👉 **Journey Builder** → Applies logic  
👉 **Studios** → Send communication  

---

### 💡 Key Takeaway
👉 **Studios = Execution**  
👉 **Builders = Logic + Data**  

✅ Understanding this = Strong SFMC foundation
