## 🚀 Journey Builder Entry Sources (All Types Explained)

---

### 🚨 Reality Check
Most people think Journey starts only from a **Data Extension**…  
👉 That’s where they lose interviews in Salesforce Marketing Cloud.  

**Entry Source = How data enters your journey**  
👉 Wrong choice = wrong customer experience  

---

### 🔍 1️⃣ Data Extension Entry (Batch Entry)
Contacts enter from a **Data Extension**  
Usually triggered via **Automation Studio**  

👉 **Use When:**  
✔️ Bulk campaigns  
✔️ Scheduled journeys  
✔️ Daily/weekly data loads  

👉 **Type:** Batch (Not Real-Time)  

---

### ⚡ 2️⃣ API Event Entry (Real-Time Power)
Triggered via **API call**  
Instant journey entry  

👉 **Use When:**  
✔️ User actions (signup, purchase)  
✔️ App/Web events  
✔️ Real-time personalization  

👉 **Type:** Real-Time  

---

### 🔗 3️⃣ Salesforce Data Entry
Triggered from **Sales/Service Cloud events**  

**Examples:**  
✔️ Lead created  
✔️ Case updated  
✔️ Opportunity closed  

👉 **Use When:**  
✔️ CRM-driven journeys  
✔️ Sales + Marketing alignment  

👉 **Type:** Near Real-Time  

---

### 🌐 4️⃣ CloudPages Entry
Triggered via **form submission**  

👉 **Use When:**  
✔️ Lead capture forms  
✔️ Newsletter signup  
✔️ Event registration  

👉 **Type:** Real-Time  

---

### 🎯 When to Use What (Real Project Thinking)
- **Use Data Extension** → Bulk campaigns / scheduled marketing  
- **Use API Event** → Instant triggers (purchase, login)  
- **Use Salesforce Data** → CRM-driven journeys  
- **Use CloudPages** → Form-based journeys  

---

### 💡 Pro Insight
Entry Source defines your **journey speed**.  
👉 Not all journeys should be real-time.
